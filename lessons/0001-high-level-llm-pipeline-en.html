Lesson 1

High-Level Intuition of the LLM Pipeline

How is a sentence processed inside the model to produce output?

The Goal of an LLM

Large Language Models (LLMs) like ChatGPT, Gemini, or LLaMA are essentially extremely powerful "next word estimators" or a Next Token Predictor.

The model's job is simple: you give it some input text (called a Prompt), and it calculates using math and statistics which word or token is most likely to follow that text. This process is called Autoregressive Generation, because the model feeds its own previously generated words back as input to continuously predict the next word.

This entire magic stands on the foundation of the Transformer Architecture.

The Core Pipeline (4 Core Steps)

Here are the 4 main steps that outline how a prompt is processed through the model:

1

Tokenization

Computers do not understand language or words directly. Therefore, the input text is broken down into smaller pieces called Tokens. Each token is assigned a unique number or Token ID.

2

Embedding

Each Token ID is converted into a High-dimensional Vector (a list of numbers). This vector captures the grammatical and semantic relationships of the words. For example, the vectors for "king" and "queen" will be close to each other in this space.

3

Transformer Blocks

This is the brain of the model. Inside lies the Self-Attention Mechanism. It allows each token to look at other tokens around it and determine how they relate to one another (Context Extraction).

4

Output & Softmax (Next Token Prediction)

At the final layer of the Transformer, a probability score is computed for every word in the model's entire vocabulary. The Softmax function converts these scores into probabilities (between 0 and 1), and the token with the highest probability is selected.
